## Findings

### [WARN] Tier 1 parser bug is float-splitting, not Unicode
**Category:** bug
**Description:** `extract_tier1()` in `run_spike.sh` uses `\b(\d+)\b` regex which splits `94.7` into `['94', '7']`, making `nums[3]` = `'7'` instead of the domain-correct value at `nums[4]`. The findings document attributes the failure to "Unicode box-drawing characters" — the actual root cause is this float-splitting.
**Evidence:** `make eval-profile | python3 -c "..."` produces `180/190 7/190` — domain field is wrong.
**Recommendation:** Fix regex to skip floats: `re.findall(r'(?<!\d\.)\b(\d+)\b(?!\.\d)', line)`. Update findings doc root cause description. Low priority since Tier 1 recheck is already flagged as a prerequisite for production promotion.

### [INFO] --exclude-column-types flag not exposed as CLI arg
**Category:** test-gap
**Description:** Spec S-1 mentions `--exclude-column-types` flag. Implementation hardcodes both variants (with/without column types) unconditionally.
**Evidence:** No `--exclude-column-types` in argparse block of `prepare_spike_data.py`.
**Recommendation:** Acceptable simplification for a spike — script correctly produces both datasets.

### [INFO] comparison.csv format doesn't match run_spike.sh header
**Category:** environment-mismatch
**Description:** The CSV on disk was regenerated by a cleanup script with different columns than `run_spike.sh` would produce. CSV is gitignored; JSON files are authoritative.
**Recommendation:** No action needed — JSON files are the source of truth.

### [INFO] Missing pre-flight checks for required inputs
**Category:** edge-case
**Description:** `prepare_spike_data.py` gives raw FileNotFoundError/traceback on missing distilled file or finetype binary.
**Recommendation:** Add explicit checks at top of `main()`. Low priority for spike tooling.
