## Findings

### [WARN] Scoring is non-deterministic (~0.2% variance)
**Category:** spec-compliance
**Description:** Scoring produces slightly different results across runs (1,934, 1,935, 1,937 across four runs). FineType's column-level inference pipeline samples values internally, introducing variance. Baseline.md claims 1,933.
**Evidence:** Four consecutive runs of `score_tier2.py` yielded different `overall_correct` values.
**Recommendation:** Document non-determinism in baseline.md (e.g., "77.6% ±0.2%"). Not a blocker — variance is small and this is a measurement tool.

### [INFO] Spec claims 172 distilled types; actual is 122
**Category:** spec-compliance
**Description:** After ≥5 values filter and taxonomy matching, only 122 types contribute distilled columns (62 fully + 60 partial). The 172 figure counts all types with any final_label.
**Recommendation:** Update spec constraint to reflect 122, or note the filtering. Cosmetic.

### [INFO] No graceful error on missing input files
**Category:** edge-case
**Description:** Both scripts crash with raw Python tracebacks when inputs missing.
**Recommendation:** Add try/except with user-friendly errors. Not a blocker — developer scripts.

### [INFO] identity.person.password missing (249/250 types)
**Category:** spec-compliance
**Description:** No generator, no distilled data. Documented in baseline.md.
**Recommendation:** Update AC-1 to "249 of 250 types" or add a password generator.
