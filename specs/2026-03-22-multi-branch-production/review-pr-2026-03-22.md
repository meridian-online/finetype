## Findings

### [CRITICAL] eval.sh/profile_eval.sh don't pass --model-type flag
**Category:** environment-mismatch
**Description:** `eval.sh` re-points `models/default` symlink but `profile_eval.sh` runs `finetype profile` without `--model-type`. Default is `char-cnn`, so it will try to load multi-branch weights as CharCNN and fail. The M-1 verification procedure cannot succeed as written.
**Evidence:** `eval/profile_eval.sh` and `scripts/eval.sh` not modified in diff. No `model.type` handling in either script.
**Recommendation:** Add `FINETYPE_MODEL_TYPE` env var support to `profile_eval.sh` and auto-detect in `eval.sh`.

### [WARN] NoopClassifier reachable via --no-header-hint + --model-type multi-branch
**Category:** bug
**Description:** `classify_column()` calls `self.classifier.classify_batch()` unconditionally. When `--no-header-hint` is passed with multi-branch, `classify_column` is called instead of `classify_column_with_header`, hitting the NoopClassifier.
**Evidence:** `column.rs:591-620` — no multi-branch guard. `main.rs:3697-3698` — `no_header_hint` branch calls `classify_column`.
**Recommendation:** Add multi-branch early-return in `classify_column()` mirroring `classify_column_with_header()`.

### [WARN] CWD-relative Model2Vec path in multi_branch.rs
**Category:** bug
**Description:** `PathBuf::from("models/model2vec")` only works from project root. No embedded-bytes fallback for release binaries (unlike existing `load_semantic_hint()`).
**Evidence:** `multi_branch.rs:215`. No `embedded` or `from_bytes` in the file.
**Recommendation:** Acceptable for M-1 (dev/eval only). Must fix before M-8 (production).

### [WARN] Potential NaN panic in argmax
**Category:** edge-case
**Description:** `.partial_cmp(b).unwrap()` at `multi_branch.rs:272` will panic if forward pass produces NaN.
**Recommendation:** Use `.unwrap_or(std::cmp::Ordering::Equal)`.

### [INFO] No integration test with real model artifact
**Category:** test-gap
**Description:** Two unit tests cover config parsing and dir detection. No test exercises the full load→classify path.
**Recommendation:** Minimal fixture test with tiny model.safetensors would close the gap.

### [INFO] _header unused in classify_multi_branch
**Category:** code-quality
**Description:** Header is intentionally unused (Sharpen deferred to M-8). `disambiguation_rule` set to `"multi-branch"` — will appear in JSON output. Track for M-8.

## Honest Assessment

The Rust implementation is structurally sound — all four M-1 changes are present, the existing CharCNN path shows no regression, code compiles with zero warnings, and 527 tests pass. The critical gap is that the M-1 verification procedure is unexecutable: `eval.sh` doesn't pass `--model-type` and the `--no-header-hint` edge case hits NoopClassifier. Two changes needed: (1) update eval scripts to forward model-type, and (2) guard `classify_column()` against multi-branch.
