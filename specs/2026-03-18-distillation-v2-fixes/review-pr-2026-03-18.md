## Findings

### [CRITICAL] CI will fail: clippy errors in detect_epoch_seconds
**Category:** bug
**Description:** `cargo clippy -- -D warnings` produces two errors: "manual `RangeInclusive::contains` implementation" on the epoch range checks.
**Evidence:** Lines ~3127 and ~3129 in column.rs: `n >= EPOCH_MIN && n <= EPOCH_MAX`
**Recommendation:** Replace with `(EPOCH_MIN..=EPOCH_MAX).contains(&n)` and same for milliseconds.

### [WARN] No unit tests for new boolean override paths
**Category:** test-gap
**Description:** `boolean_override_single_value` and `boolean_override_skewed` have no direct unit tests.
**Recommendation:** Add tests calling `disambiguate_boolean_override` with all-zero and 95%-skewed inputs.

### [WARN] No unit tests for detect_epoch_seconds
**Category:** test-gap
**Description:** No tests for the epoch detection function — in-range, out-of-range, threshold boundary, milliseconds.
**Recommendation:** Add unit tests covering all paths.

### [WARN] Financial guard blocks legitimate financial headers
**Category:** edge-case
**Description:** Header "basis_points" with actual financial values (0.0025) classifies as latitude because the guard blocks ALL finance.* model2vec hints when CharCNN votes non-financial. Net positive per distillation data.
**Recommendation:** Track as known gap. Consider value-pattern co-occurrence guard as future refinement.

### [INFO] Pre-2000 epoch timestamps not covered
**Category:** edge-case
**Description:** EPOCH_MIN is 2000-01-01. 1990s timestamps fall through.
**Recommendation:** Track as known gap for Phase 3.

### [INFO] Dominant count formula in boolean_override could be clearer
**Category:** edge-case
**Description:** `.max(count(v != parsed[0]))` is correct but confusing.
**Recommendation:** Add clarifying comment.
