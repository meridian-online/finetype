### [WARN] Stats branch is 27-dim, spec says 36-dim
**Category:** test-gap (spec violation)
**Description:** Spec says `stats: 36→128→64` and `stat_features ([f32; 36])` and `960 + 512 + 36 = 1508`. Implementation outputs 27-dim consistently (Rust, Python, FTMB format). Total is 1499, not 1508.
**Recommendation:** Document the deviation. Update spec or add 9 more stats features to reach 36.

### [WARN] No architecture summary printed (AC-4 criterion)
**Category:** test-gap
**Description:** AC-4 requires "Architecture summary printed shows correct input/hidden/output dimensions per branch." Only parameter count is logged, not branch dimensions.
**Recommendation:** Add per-branch dimension logging at training start.

### [WARN] Docstring says bytes 32-126, implementation uses 32-127
**Category:** bug (documentation)
**Description:** Char distribution docs say "bytes 32-126" but code includes byte 127 (DEL). DEL is the 96th character to hit the 960-dim target.
**Recommendation:** Fix docstring to say "bytes 32-127 (space through DEL)."

### [INFO] Empty string input produces zeros (not error)
**Category:** edge-case
**Description:** `echo "" | finetype extract-features` exits 0 with all-zero features. Technically correct.
